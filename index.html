  150. The easiest way to change your program icons individually, without any tools or software, is to edit a shortcut. What I recommend doing is navigating to the programs root directory, and creating the shortcut there so there is no confusion. Creating a shortcut also gives you the added bonus of naming it whatever you want, and even setting launch options. Right click on the Icon>Properties>Change Icon. Use your desired .ico file. Pin this shortcut to your taskbar/start menu! For other icons (library icons, task tray icons) please see the windows software section for tools to help you change them. Its a lot faster and easier than going through registries and replacing files manually. Look for 7tsp or 7conifier for icon packs, and browse for icon packs in the Icon section!

  167. -Making sure your taskbar is aligned properly, and slim as possible-
  168. Ever install appows and notice the icons or the start orb seem to be a little misaligned? This is just a glitch with the taskbar that is easily solved by extending it. Unlock your task bar, and drag it to the top of your screen. You can do this by left clicking and holding the bar near the center, and moving it to the edges of the screen. Drag it to the top along the edges this way. Once its on top, hover over the middle of the bar near the bottom border until you see the resize option for height ?. Drag it down, extending it to double the normal height. Move it back down to the bottom of your screen, then size it back down using the same ? option. Lock your task bar. You should be at the correct size and alignment now.

  334. ***Font Rendering***
  335. Fonts appearing to have a greenish tint or look blurry in Firefox/Bblean/Windows? This is due to an acceleration error, probably caused by subpixel rendering or fonts not being supported in smaller sizes. Windows Cleartype also causes issues with EVERY browser, and overall is complete shit compared to Linux or Mac/OSX font rendering. Read about Cleartype for details on how it works. Consider switching fonts if none of the fixes below work for you, or try and get a version of your font that is NOT ".tff" or true type.
  336.  
  337. The best and easiest option in my opinion is to install Mactype or gdipp. These are services that replace Windows Cleartype font rendering with Mac/OSX or Linux font rendering. Mactype is more customizable and has bolder bolds, while Gdipp is easier to use and works immediately upon install. If you are using a bitmap font, font rendering is not necessary. Cleartype should be off and you should not need to use Mactype or gdipp. This is only true if you are using only bitmap fonts. If you are using a mix of fonts, I recommend Mactype since its easier to edit your exclusions.
  338. Go here for details.
  339.  
  340. Now you can compare programming font rendering live on your webpage with this font compare tool! See the difference between cleartype, mactype and gdipp with your own eyes.
  341.  
  342. MacType is now open source and receiving updates! New releases are available at: snowie2000s repo, and on the project's homepage.
  343. Having issues with mactype and direct write? This may help: silight-jp MacType-Patch
  344.  
  345. Recommended fixes for Firefox/Nightly:
  346. In Firefox, type "about:config" in the address bar and hit Enter. Continue after the warning.
  347. In the filter box at the top, type "gfx.direct2d.disabled". This value will be set to false by default.
  348. Right click that line and select Toggle. This will change the value to true. Restart Firefox. You may also want to toggle "Use hardware acceleration when available" in your Firefox settings. This is under Options>Advanced>General tab.
  349. Try the Anti-Aliasing Tuner too, if you feel the need for more tweaking.

This method is the easiest way to not only change all the fonts in your visual style, but ensure they will render at the best quality possible. If you choose to use a sans font, stick with the recommended fonts below and use LCD Default settings with MacType for best results.   405. ***Cygwin***
  406.  
  407. Official Site - FAQ
  408.  
  409. (From wikipedia) "Cygwin is a Unix-like environment and command-line interface for Microsoft Windows. Cygwin provides native integration of Windows-based applications, data, and other system resources with applications, software tools, and data of the Unix-like environment. Thus it is possible to launch Windows applications from the Cygwin environment, as well as to use Cygwin tools and applications within the Windows operating context."
  410.  
  411. Before you dive into Cygwin, you should ask yourself a few questions.
  412. "Do I have a basic understanding of how the Unix environment works?"
  413. "Am I literate enough in bash, and other scripting languages to understand how things work?"
  414. "Do I have any practical application or uses for what Cygwin can offer me?"

  795. ***Tiling Windows***
  796. If you'd like to make your own AHK Script to manage window positions and sizes, use this key as reference:
  797. "WinMove,A,, W,X,Y,Z"
  798. Where
  799. W= Distance of the window from the left side of the screen
  800. X= Distance of the window from the top of the screen
  801. Y= Total width of the window
  802. Z= Total height of the window
  803.  
  804. Like so:
  805. NumpadHome:: (defined hotkey)
  806.             WinMove,A,, 20,29,1024,512 (defined window size and position)
